# OlfaBind 시행착오 기록

이 문서는 OlfaBind 모델 연구 과정에서 시도한 접근법, 실패 원인, 그리고 각 실패로부터 얻은 교훈을 기록한다. 코드 수준의 오류는 제외하고, 연구 방향성과 관련된 실험적 시행착오만 다룬다.

---

## 1단계: 대조 학습 도입 시도 (v19~v21)

### 배경

v18에서 물리 엔진 baseline(r=0.680)을 확립한 뒤, 대조 학습(Contrastive Learning)을 통해 물리 엔진의 초기 좌표를 더 의미 있게 학습시키면 성능이 향상될 것이라 가정하였다. SimCLR, MoCo 등 자기 지도 학습의 성공 사례에서 영감을 얻었다.

### v19: InfoNCE 손실 함수

시도: SimCLR 스타일 InfoNCE 손실 함수를 물리 엔진과 결합. 같은 혼합물의 두 augmented view를 positive pair로, 배치 내 다른 혼합물을 negative pair로 사용.

결과: r=0.594 (baseline 대비 -0.086)

실패 원인 분석:
- 대조 학습의 목표(같은 혼합물의 표현을 가깝게)와 유사도 예측의 목표(두 혼합물 간 연속적 유사도 예측)가 근본적으로 상충한다.
- 대조 학습은 이진 분류(같다/다르다)에 최적화되어 있어, 연속적 유사도 회귀 문제에 적합하지 않다.
- InfoNCE의 temperature(0.07)가 물리 엔진의 좌표 공간 스케일과 맞지 않아 gradient가 과도하게 발생하였다.

교훈: 대조 학습의 목표 함수와 최종 과제(유사도 예측)의 목표 함수가 일치하는지 반드시 확인해야 한다.

### v20: Triplet Margin Loss

시도: InfoNCE를 Triplet Margin Loss로 대체. anchor-positive-negative 삼중항 구조를 사용하여 margin 기반 학습.

결과: r=0.436 (baseline 대비 -0.244, 최악)

실패 원인 분석:
- Snitz 데이터셋(360 쌍)은 대조 학습에 필요한 데이터 규모에 크게 미달한다. 일반적으로 대조 학습은 수만~수십만 샘플이 필요하다.
- margin=1.0이 데이터의 자연적 분포를 고려하지 않아, 과도한 분리를 강제하여 표현 공간이 붕괴(collapse)되었다.
- in-batch negative sampling에서 진정한 negative(실제로 다른 혼합물)의 비율이 낮아, false negative 문제가 발생하였다.

교훈: 데이터셋 규모가 작은 경우(수백 샘플) 대조 학습은 근본적으로 적용이 어렵다. margin 값은 데이터 분포에 맞게 adaptive하게 조정되어야 한다.

### v21: 6개 기법 추가

시도: v20의 문제를 해결하기 위해 6가지 개선을 동시에 적용.
- 적응적 margin (학습 가능한 nn.Parameter)
- Hard negative mining (가장 어려운 negative 우선 선택)
- Similarity-weighted loss (ground truth 유사도를 가중치로 사용)
- Layer-wise learning rate (물리 엔진 1e-5, head 5e-4)
- Projection head 개선 (128D 중간층 + LayerNorm)
- Validation-aware weighting (검증 성능 기반 동적 loss 가중치)

결과: r=0.553 (v20 대비 +0.117, 하지만 여전히 baseline 미달)

실패 원인 분석:
- 6개 기법을 동시에 추가하여 어떤 것이 효과적이고 어떤 것이 역효과인지 분리가 불가능하였다 (ablation 부재).
- 근본적인 문제(대조 학습 목표와 유사도 목표의 상충)가 해결되지 않은 채 기법만 추가한 것은 표면적 처방에 불과하였다.
- 학습 가능한 하이퍼파라미터(adaptive margin 등)가 오히려 학습 불안정성을 증가시켰다.

교훈: 근본 문제를 해결하지 않고 기법만 누적하는 것은 효과가 제한적이다. 여러 기법을 동시에 적용할 때는 반드시 ablation study를 병행해야 한다.

## 2단계: 물리 기반 손실 함수 (v22)

### 배경

대조 학습이 실패한 원인이 목표 함수의 상충이라고 진단한 뒤, 물리학 논문(Hamiltonian Neural Networks, Physics-Informed Neural Networks)에서 영감을 얻어 물리 법칙 자체를 손실 함수로 사용하는 접근법을 시도하였다.

### 시도

3가지 물리 기반 손실 함수를 동시에 적용:
- Hamiltonian Trajectory Matching: 해밀턴 역학의 에너지 보존 법칙을 시간 스텝마다 정규화
- PINN Regularization: 뉴턴의 운동 방정식 F=ma의 잔차(residual)를 최소화
- Spectral Matching: 궤적의 스펙트럼 분해를 통한 주파수 영역 정규화

### 결과: r=0.532 (baseline 대비 -0.148)

### 실패 원인 분석
- 물리 법칙 준수와 유사도 예측 성능은 독립적이다. 에너지가 잘 보존된다고 해서 두 혼합물의 유사도를 더 잘 예측하는 것은 아니다.
- 물리 손실이 gradient를 지배하여 MSE(유사도) 손실의 학습을 방해하였다. 두 손실 간의 gradient 경쟁(gradient conflict)이 발생.
- 25 folds 중 일부에서 r>0.67 (baseline급)이 관찰되었으나, 다른 folds에서 r<0.3이 나오며 변동성이 매우 높았다 (std=0.119).

### 교훈
물리적으로 올바른 시뮬레이션이 반드시 판별력 있는 특징을 생산하는 것은 아니다. 물리 법칙은 보편적이어서, 오히려 혼합물 간의 차이를 줄이는 방향으로 작용할 수 있다.

## 3단계: 자유도 극대화 + 안정성 강화 (v23)

### 배경

v19~v22의 실패가 모델의 자유도(degree of freedom)가 부족하여 발생한 것이라는 가설을 세우고, 자유도와 안정성을 동시에 극대화하는 전략을 설계하였다.

### 시도

자유도 확대:
- Multi-scale Simulation: T=2, 4, 8 세 가지 시간 스케일의 시뮬레이션을 동시 실행하여 다중 해상도 물리 특징을 추출
- Trajectory Attention: 수작업 물리 특징 대신 학습된 어텐션 메커니즘으로 궤적에서 특징 추출
- 8D Latent Space: 3D 위치 공간을 8D로 확장하고 Position Adapter로 물리 엔진 입력 변환

안정성 강화:
- 10-restart (3에서 증가)
- SWA (Stochastic Weight Averaging)
- Mixup 데이터 증강
- Warmup + Cosine Decay 스케줄러

### 결과: r=0.520 (baseline 대비 -0.160)

### 실패 원인 분석
- Multi-scale이 fold당 학습 시간을 3배 증가시켰으나, 세 스케일의 특징이 서로 상충하여 오히려 노이즈를 생성하였다.
- 8D Latent Space에서 3D로의 Position Adapter가 병목이 되어 8D의 추가 자유도가 물리 엔진에 전달되지 못했다.
- Trajectory Attention이 학습 초기에 불안정하여 좋은 초기화 없이는 수렴이 어려웠다.
- 최고 folds(0.709, 0.687)는 baseline을 초과하였으나, 최저 folds(0.282, 0.285)가 평균을 크게 끌어내렸다. 이는 추가 구성요소가 국소 최솟값의 수를 증가시킨다는 것을 의미한다.

### 교훈
복잡도를 높이면 잠재력(상한)은 올라가지만 안정성(하한)은 내려간다. 작은 데이터셋에서는 복잡한 모델이 과적합이 아닌 과소적합(underfitting)의 형태로 실패할 수 있다. 모든 추가 구성요소는 추가 국소 최솟값을 만들어 restart의 효과를 희석시킨다.

## 4단계: 내부 구조 보수적 개선 (v24)

### 배경

v19~v23의 교훈을 종합하여, 물리 엔진 위에 모듈을 추가하는 것이 아니라 내부 구조만 개선하는 보수적 접근을 시도하였다.

### 시도
- ConstellationToCelestial 매퍼를 단순 Linear에서 2-layer MLP + LayerNorm + Residual로 교체
- 물리 임베딩을 20D에서 32D로 확장 (궤적 분산, 쌍별 거리 통계, 질량 중심 이동, 에너지 비율 등 12개 특징 추가)
- Projection head에 Dropout(0.1) 추가

### 결과: r~0.44 (baseline 대비 -0.24, 8/25 folds에서 중지)

### 실패 원인 분석
- 단순 Linear 매퍼가 이미 최적이었다. MLP로의 교체가 오히려 매핑의 일관성을 해쳤다.
- Linear의 직접 매핑은 gradient가 깨끗하게 전파되지만, MLP의 비선형 계층은 gradient 경로를 복잡하게 만들어 물리 엔진까지의 학습 신호 전달을 방해하였다.
- 추가된 12개 물리 특징이 원래 20D 특징과 스케일이 달라(예: 에너지 비율은 O(1), 궤적 분산은 O(0.01)) 정규화 없이 단순 concat한 것이 문제가 되었다.
- Residual connection이 원래 constellation 정보를 과도하게 유지하여 매핑의 변환력을 약화시켰다.

### 교훈
잘 작동하는 단순한 구조를 무분별하게 복잡화하면 안 된다. 특히 Linear 매핑은 gradient 전파가 깨끗하다는 고유한 장점이 있어, MLP로의 교체가 반드시 개선이 되는 것은 아니다. 이것은 물리 시뮬레이션을 포함하는 파이프라인에서 특히 중요하다.

## 5단계: 학습 전략만 최적화 (v25)

### 배경

v19~v24에서 구조 변경이 모두 실패한 것을 확인한 뒤, 구조를 완전히 고정하고 학습 전략, 데이터, 하이퍼파라미터만 최적화하는 접근법으로 전환하였다.

### Strategy A: 학습 전략 최적화

시도: v18 원본 구조에 10-restart, SWA, warmup(5 epochs), cosine decay, gradient accumulation(effective batch=64), gradient clipping(0.5) 적용.

결과: r=0.727

분석:
- 10-restart가 가장 큰 기여. 손실 지형의 다수 국소 최솟값 중 더 좋은 것을 선택할 확률이 높아짐.
- SWA와 warmup은 제한적 기여. 오히려 SWA가 특정 folds에서 최적 모델을 평균으로 희석시키는 경우가 관찰됨.
- gradient accumulation(effective batch=64)은 작은 데이터셋에서 배치 통계의 안정성을 높였으나, 학습 속도를 4배 느리게 만듦.

### Strategy B: 데이터 증강

시도: Bushdid 2014 discrimination 데이터를 Jaccard 유사도 기반 pseudo-similarity로 변환하여 1,046개 추가 학습 쌍 생성. 원래 Snitz 360쌍에 추가하여 학습.

결과: r=0.565 (baseline 대비 -0.115)

실패 원인 분석:
- Bushdid 데이터는 discrimination(구별 가능/불가능)이지 perceptual similarity(지각적 유사도)가 아니다. 두 과제의 특성이 근본적으로 다르다.
- Jaccard 유사도(구성 분자의 중복률)를 지각적 유사도의 대리 지표로 사용한 것이 부정확하여, 잘못된 레이블이 학습을 오염시켰다.
- 추가 데이터의 분포가 원래 Snitz 데이터와 달라(도메인 시프트) 일반화 성능이 오히려 하락.

교훈: 단순히 데이터 양을 늘리는 것은 데이터의 질이 보장되지 않으면 역효과를 낳는다. 도메인이 다른 데이터를 무분별하게 추가하면 분포 불일치 문제가 발생한다.

### Strategy C: 하이퍼파라미터 탐색

시도: 시뮬레이션 길이(T)와 시간 간격(dt)의 7가지 조합을 grid search.

결과:
- T=2, dt=0.1: r=0.729
- T=4, dt=0.05: r=0.729 (현재 기본값과 동일)
- T=8, dt=0.025: r=0.461
- T=16, dt=0.0125: r=0.404
- T=4, dt=0.1: r=0.686
- T=4, dt=0.025: 별도 측정
- T=8, dt=0.05: 별도 측정

분석:
- T=2와 T=4가 거의 동일한 성능을 보여, 4스텝의 짧은 시뮬레이션으로도 충분한 판별력을 가진 물리 특징이 추출됨을 확인.
- T가 8 이상으로 증가하면 성능이 급격히 하락. 긴 시뮬레이션은 수치적 불안정(numerical instability)을 증가시키고, 초기 조건의 미세한 차이가 발산하여 혼합물 간 판별력이 사라진다.
- dt가 너무 크면(0.1) 시뮬레이션의 정확도가 떨어지고, 너무 작으면(0.025) 4스텝 내에 충분한 궤도 변화가 발생하지 않아 모든 혼합물이 비슷한 궤적을 보인다.

### Baseline (원본 구조, 원본 학습, 10-restart)

결과: r=0.780

분석:
- 놀랍게도 SWA, warmup 등의 기법을 적용한 Strategy A(0.727)보다 원본 학습 방식(단순 Adam + CosineAnnealing)에 restart만 늘린 것이 더 높다.
- SWA가 일부 folds에서 최적 모델의 성능을 평균으로 희석시키는 역효과를 낸 것으로 추정. 특히 손실 지형이 다봉(multi-modal)인 경우, 여러 모델을 평균하면 각 모드의 장점이 상쇄될 수 있다.
- 원본 Adam optimizer의 적응적 학습률이 이미 충분히 효과적이어서, 추가적인 스케줄링 기법이 불필요하였다.

### 교훈
가장 강력한 최적화 전략은 가장 단순한 것이었다. restart 횟수만 3에서 10으로 증가시킨 것이 어떤 정교한 학습 기법보다 효과적이었다. 이는 모델의 손실 지형에 다수의 국소 최솟값이 존재하고, 좋은 국소 최솟값을 찾는 확률을 높이는 것이 가장 직접적인 해결책임을 보여준다.

## 전체 시행착오 흐름 요약

```
v18 baseline (r=0.680)
│
├─ v19: 대조 학습 도입 → 실패 (목표 함수 상충)
│   └─ v20: Triplet로 변경 → 더 나빠짐 (데이터 부족)
│       └─ v21: 6개 기법 추가 → 미미한 개선 (ablation 부재)
│
├─ v22: 물리 기반 손실 → 실패 (물리 법칙 ≠ 판별력)
│
├─ v23: 자유도 극대화 → 실패 (복잡도가 안정성 파괴)
│
├─ v24: 내부 구조 개선 → 실패 (단순 Linear가 최적)
│
└─ v25: 학습 전략만 최적화 → 성공
    ├─ A: SWA+warmup → r=0.727 (일부 효과)
    ├─ B: 외부 데이터 → r=0.565 (역효과)
    ├─ C: HP search → r=0.732 (일부 효과)
    └─ Baseline+10-restart → r=0.780 (가장 단순, 가장 효과적)
```

### 이 연구에서 얻은 근본적 교훈

1. 작동하는 단순한 구조를 존중하라. 복잡화가 반드시 개선을 의미하지 않는다.

2. 데이터가 작을 때 모델을 키우지 마라. 360쌍의 데이터로는 대조 학습, 물리 손실, 어텐션 등 대규모 데이터에서 효과적인 기법이 작동하지 않는다.

3. 최적화 전략은 문제에 맞게 선택하라. 다봉 손실 지형에서는 정교한 gradient 기반 기법(SWA, warmup)보다 탐색 범위를 넓히는 기법(multi-restart)이 효과적이다.

4. 물리적 정확성과 과제 성능은 별개이다. 물리 법칙을 더 잘 따르는 시뮬레이션이 반드시 더 좋은 특징을 생산하는 것은 아니다.

5. 도메인이 다른 데이터의 추가는 신중해야 한다. 양의 증가보다 질의 일관성이 중요하다.
